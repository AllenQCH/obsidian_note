# 01-01-LLM-总览

## 摘要

- LLM 是负责理解、推理、决策和生成文本的通用推理引擎。
- LLM 本身没有 IO、执行环境和长期记忆；要形成可用系统，通常需要 [[02-01-RAG-总览]]、[[Tool]]、[[01-06-LLM-Memory]] 和 [[00-agent introduction|Agent]] 配合。
- LLM 基础知识可以按“模型 -> Token -> Prompt -> Context -> 参数 -> Memory”的顺序学习。

## 核心内容

### LLM 在系统里的位置

```text
LLM：负责理解、推理和生成
Prompt：告诉 LLM 当前任务和约束
Context：LLM 当前能看到的全部材料
RAG：把外部知识动态放进 Context
Tool：让 LLM 通过系统调用外部能力
Agent：组织多步决策和工具调用流程
```
