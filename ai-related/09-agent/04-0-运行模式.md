这是**Agent 的运行模式 / 策略模式**，不是平台，也不是框架。
ReAct / Plan-and-Execute 是“做的时候按什么思路跑”
ReAct、Plan-and-Execute 是 Agent 内部常用的执行模式。

# 04-0-运行模式

## 摘要

Agent 运行模式描述的是：**Agent 按照什么策略组织推理、行动、观察、规划和重规划。**

它不是模型能力，也不是框架或平台。Reasoning、Memory、Tool Calling 是 Agent 可使用的能力；ReAct、Plan-and-Execute 则负责组织这些能力的运行方式。

## 核心内容

| 运行模式 | 核心方式 | 适用任务 |
| --- | --- | --- |
| [[04-1-ReAct]] | Reason → Action → Observation，边做边调整 | 开放探索、信息不完整、工具结果决定下一步 |
| [[04-2-Plan-and-Execute]] | 先整体规划，再按步骤执行，必要时重规划 | 步骤较多、目标清晰、需要全局协调的长任务 |

实际 Agent 可以混合使用：先通过 Plan-and-Execute 形成高层计划，再让每一个执行步骤内部采用 ReAct。

## 可执行动作

设计 Agent 前先判断：

1. 路径是否需要根据环境反馈动态变化？优先考虑 ReAct。
2. 是否需要先形成全局任务分解？优先考虑 Plan-and-Execute。
3. 是否同时需要全局规划和局部探索？采用混合模式。
4. 无论采用哪种模式，都要设置状态、最大步数、权限、失败处理和终止条件。
