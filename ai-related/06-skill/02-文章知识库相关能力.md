# 02-文章知识库相关能力

## 摘要

这篇笔记整理 4 种后续会高频用到的能力：**llm-wiki、obsidian、ocr-and-documents、blogwatcher**。它们分别解决“长期沉淀、写笔记、提文本、追更新”四类问题。组合起来，可以形成一套完整的文章知识库工作流。

## 1. 四种能力的定位

### 1. llm-wiki
- **适合**：长期读文章、持续积累、做知识库
- **能力**：把文章 ingest 进 markdown wiki，做交叉引用、更新索引、沉淀成长期笔记
- **适用场景**：看一批 AI / 工程文章，然后持续整理观点、方法论、时间线

### 2. obsidian
- **适合**：把文章整理成笔记
- **能力**：读 / 搜 / 写 Obsidian 里的 markdown 笔记
- **适用场景**：已经有现成 Obsidian vault，希望所有内容都沉淀到自己的知识库里

### 3. ocr-and-documents
- **适合**：文章是 PDF、扫描件、截图、文档
- **能力**：抽文本、OCR、转 markdown
- **适用场景**：先把文档内容提出来，再交给我总结、归档、写入知识库

### 4. blogwatcher
- **适合**：持续追踪博客更新
- **能力**：订阅博客 / RSS，扫描新文章
- **适用场景**：长期跟 Anthropic / OpenAI / LangChain / Martin Fowler 这类站点更新

## 2. 这 4 个能力分别解决什么问题

### llm-wiki 解决：内容如何长期沉淀
如果只是零散记笔记，后面很容易变成“收藏夹堆积”。

llm-wiki 的价值在于：
- 不只是记一篇文章
- 而是把文章放进一个可增长的 markdown wiki
- 让不同文章之间能形成：
  - 时间线
  - 主题页
  - 实体页
  - 对比页
  - 查询页

一句话：
**llm-wiki 解决“知识如何越积越值钱”。**

### obsidian 解决：知识如何放进自己的系统
Obsidian 的价值在于：
- 你的知识库在本地
- markdown 可控
- 双链方便
- 搜索和回顾体验好

一句话：
**obsidian 解决“知识最后放哪、怎么持续维护”。**

### ocr-and-documents 解决：内容怎么拿出来
很多材料不是干净网页，而是：
- PDF
- 长截图
- 扫描版
- 分享文档

这时如果没有 OCR / 提取能力，后续总结和归档就很痛苦。

一句话：
**ocr-and-documents 解决“原始内容怎么变成可处理文本”。**

### blogwatcher 解决：后续更新怎么不断掉
知识库不是一次性工程，而是持续更新的。

blogwatcher 的价值在于：
- 盯住博客源
- 有新文章就发现
- 再进入“提取 → 总结 → 写入知识库”的流程

一句话：
**blogwatcher 解决“新内容怎么持续进入系统”。**

## 3. 推荐组合方式

### 组合 A：最小可用版
- **obsidian**
- **ocr-and-documents**

适合：
- 先把文章、截图、PDF 都整理成笔记
- 不追求复杂 wiki 结构

### 组合 B：长期知识库版
- **llm-wiki**
- **obsidian**
- **ocr-and-documents**

适合：
- 想把文章变成长期可复用知识
- 不只是存链接，而是沉淀成主题、概念和方法论

### 组合 C：持续追更版
- **llm-wiki**
- **obsidian**
- **ocr-and-documents**
- **blogwatcher**

适合：
- 有固定关注源
- 希望知识库持续自动长大
- 后面做成半自动“阅读 -> 摘要 -> 归档”的 workflow

## 4. 推荐工作流

### Step 1：获取内容
- 网页文章：直接抓链接
- PDF / 截图 / 扫描件：走 [[ocr-and-documents]] 对应流程
- 博客新内容：由 blogwatcher 发现

### Step 2：提取与整理
- 提取正文
- 识别标题、作者、时间、来源
- 生成摘要、关键观点、关键词

### Step 3：沉淀到 Obsidian
- 写入具体文章卡片
- 写入主题笔记
- 加上双链、标签和时间线

### Step 4：升级为长期知识库
- 用 llm-wiki 把零散文章变成：
  - 概念页
  - 方法论页
  - 对比页
  - 时间线页

### Step 5：持续更新
- 用 blogwatcher 盯住固定博客源
- 有新文章就补进既有主题结构

## 5. 你当前最适合的用法

结合你现在的情况，最适合的是：

### 当前阶段
- **obsidian + llm-wiki 思路**
- 先把关键文章沉淀成结构化笔记
- 逐渐建立主题页、时间线和概念页

### 遇到这些情况时再加能力
- **文章是截图 / PDF / 扫描件** → 用 ocr-and-documents
- **想长期跟几个博客更新** → 加 blogwatcher

## 6. 对应到你这批 harness 文章

这 4 个能力可以这样分工：

- **obsidian**：把这 6 篇文章放进你现有 vault
- **llm-wiki**：把它们沉淀成 `harness engineering` 主题知识库
- **ocr-and-documents**：如果后面补充的是 PDF、截图、演讲截图，就先提文本
- **blogwatcher**：后面持续跟 Anthropic / OpenAI / LangChain / Martin Fowler 的新文

## 7. 一句话记忆版

- **llm-wiki**：负责“长成知识库”
- **obsidian**：负责“落到我的笔记里”
- **ocr-and-documents**：负责“把内容拿出来”
- **blogwatcher**：负责“把新内容持续送进来”
