# 当前阶段和下一步

## 摘要

当前完成的是目标架构收敛与 Obsidian 文档统一，不是运行时迁移。五类 Agent、四个 Workflow Agent、八个开发职责 Stage Agent、测试与版本交付边界已经确认；四个 Workflow 的详细阶段顺序仍待单独评审。

## 已确认

- Agent 统一命名为 `<layer>_<responsibility>_agent`。
- Router 选择 Workflow；Orchestrator 执行 Workflow 状态机。
- 当前只保留 Workflow 层，不引入独立 subflow。
- 本地联调归入 `stage_test_runner_agent`。
- 版本交付统一归入 `stage_version_delivery_agent`。
- Gate 统一由 `gate_stage_evaluator_agent` 执行。
- Skill、MCP、Script 保持能力身份，不伪装成 Agent。

## 待确认

- 四个 Workflow Agent 的逐步顺序、必需产物和每道 Gate 的规则集。
- 八个 Stage Agent 的最终 TOML prompt、输入输出 schema 和上下文隔离策略。
- 当前运行时已有 Agent 的迁移批次与兼容期。
- Tool Agent 的完整目标清单及每个 Tool 对应的 Skill/MCP/Script。

## 推荐迁移顺序

1. 评审四个 Workflow 的状态机。
2. 固化 Stage 交付物和 Gate 规则。
3. 对照 `~/.codex/config.toml` 生成运行态迁移清单。
4. 分批新增或改名 TOML，并同步全局注册表。
5. 运行目录校验、路由样例和最小端到端任务。

运行时迁移属于另一项会改动全局配置的任务，需要单独确认后执行。
