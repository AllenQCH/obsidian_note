# 当前阶段和下一步

## 摘要

当前架构已经进入维护阶段：本机运行时为 13 个 Agent，GitHub 便携版为 12 个核心 Agent，`my-openspec` 已通过统一上下文和 CLI 证据链接入。

## 已落地

- 本机运行时注册 13 个活跃 Agent，其中包括 `tool_alidocs_test_delivery_agent`。
- Workflow 不注册 Agent，由 Orchestrator 持有状态。
- 前端、产品验收、Integration Orchestrator、Closeout Reporter 已从流程移除。
- CR 阻塞回到 `stage_backend_designer`。
- `stage_code_reviewer` 已接入本机默认 CR 资料包；用户规则和项目规则优先于 fallback。
- 测试用例同时依赖产品需求和后端设计。
- 迭代再开发必须重新交付。
- Bug 排查不改代码；测试环境失败输出 `bug-list.md`。
- OpenSpec 与 Multi-Agent 分离，通过稳定字段和 CLI 衔接。

## 后续维护

只有真实重复场景出现时才新增 Workflow、Stage 或 Tool Agent。任何 Agent 变更都必须同步 TOML、`config.toml`、运行时目录、生成注册表和本目录当前入口，并重新执行 catalog 校验。
