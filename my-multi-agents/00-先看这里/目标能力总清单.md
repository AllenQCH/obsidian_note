# 目标能力总清单

## 摘要

目标体系包含 2 个 Control Agent、4 个 Workflow Agent、8 个开发职责 Stage Agent、1 个 Gate Agent，以及按能力逐步迁移的 Tool Agent。这里列的是**目标清单**，不声明已经注册生效。

## Agent 清单

### Control

- `control_request_router_agent`
- `control_stage_orchestrator_agent`

### Workflow

- `workflow_iterative_feature_development_agent`
- `workflow_rewrite_iterative_feature_development_agent`
- `workflow_bug_investigation_agent`
- `workflow_solve_personal_problem_agent`

### Stage

- `stage_product_owner_agent`
- `stage_backend_designer_agent`
- `stage_backend_developer_agent`
- `stage_frontend_developer_agent`
- `stage_test_case_designer_agent`
- `stage_code_reviewer_agent`
- `stage_test_runner_agent`
- `stage_version_delivery_agent`

补充：`stage_investigation_planner_agent` 可服务于 Bug 排查，但不计入八个开发职责 Agent。

### Gate

- `gate_stage_evaluator_agent`

### 新增 Tool

- `tool_dependency_package_publisher_agent`
- `tool_test_delivery_document_agent`

其他已有工具能力后续统一迁移为 `tool_<capability>_agent`。迁移前必须逐项核对当前注册表，不能只改文档名称。

## 非 Agent 能力

| 类型 | 识别方式 | 用途 |
| --- | --- | --- |
| Agent | TOML + `[agents.<name>]` | 有职责、上下文和输出合同的执行主体 |
| Skill | `SKILL.md` | 可复用操作规范和能力入口 |
| MCP | `[mcp_servers.<name>]` | 外部工具协议服务 |
| Script | 可执行脚本 | 底层确定性实现 |

同一个 Tool Agent 可以治理 Skill、MCP 或 Script，但不能把这些能力直接称为 Agent。
