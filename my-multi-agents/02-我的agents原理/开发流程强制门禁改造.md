# 开发流程强制门禁改造

## 摘要

门禁不是多个独立 Agent，而是一个 `gate_stage_evaluator_agent` 执行多套规则。它只读取阶段产物，输出 `go`、`warn` 或 `block`；不写代码、不补文档、不启动服务，也不替 Orchestrator 推进状态。

## Gate Agent 合同

### 输入

```yaml
workflow_id: string
current_stage: string
gate_rule: string
required_artifacts: []
evidence_paths: []
```

### 输出

```yaml
decision: go | warn | block
rule: string
missing_evidence: []
findings: []
next_stage: string | null
```

## 开发流程规则集

| 规则 | 检查重点 | `block` 示例 |
| --- | --- | --- |
| `design_confirmed` | 需求范围、设计、测试用例和未决问题 | 关键接口契约未确认 |
| `implementation_ready` | 开发产物、单测、静态检查 | 关键分支无测试或构建失败 |
| `review_passed` | CR 结论和阻断问题 | 仍有高优先级缺陷 |
| `test_passed` | 本地闭环、回归结果和失败说明 | 只启动服务但未验证业务结果 |
| `delivery_ready` | 权限、依赖、deploy 服务、提测材料 | 服务清单或依赖顺序不明确 |
| `delivery_closed` | 流水线、部署观察和归档证据 | 部署状态未知或证据未记录 |

规则名是 Gate 配置，不是 Agent ID。

## `warn` 的边界

`warn` 只用于不影响核心验收、已有明确风险接受人的非阻断项。缺失强制证据、生产风险未知、测试失败或未获得外部写入授权时必须 `block`。
