# 资源注册表总览

## 摘要

资源注册表用于回答“当前机器实际上有什么”，目标架构文档用于回答“以后要改成什么”。本页不复制资源数量，因为数量会漂移；需要当前事实时运行查询与校验。

## 真源

| 资源 | 当前真源 |
| --- | --- |
| Agent | `~/.codex/config.toml` 与对应 Agent TOML |
| Skill | 各 Skill 目录中的 `SKILL.md` |
| MCP | `~/.codex/config.toml` 的 `[mcp_servers.<name>]` |
| Plugin | Codex 配置和插件缓存元数据 |
| Script | Skill 或工具文档登记的实际脚本路径 |

## 查询入口

- HTML：`~/.codex/agent-catalog-runtime/agent_catalog.html`
- JSON：`~/.codex/agent-catalog-runtime/resource_registry.json`
- Markdown：`~/.codex/agents/docs/agent-registry.md`
- 校验：`~/.codex/agent-catalog-runtime/check_agent_catalog.sh`
- 重建：`~/.codex/agent-catalog-runtime/sync_agent_catalog.sh`

## 与目标架构的关系

- [[目标agent注册表]] 记录目标 ID 和职责。
- 本页列出的真源记录当前实际 ID 和启用状态。
- 两者不一致时，表示运行时迁移尚未完成，不应偷偷把其中一侧当成另一侧。
- 运行时迁移完成后再重建目录和登记文档。

## 查询顺序

1. 先在资源注册表查是否已有能力。
2. 再打开对应 TOML、`SKILL.md` 或 MCP 配置核对边界。
3. 涉及数据库、日志、流水线等可外部验证事实时，必须真实调用能力。
4. 只有出现稳定治理需求时，才把 Skill/MCP/Script 包装为 Tool Agent。
