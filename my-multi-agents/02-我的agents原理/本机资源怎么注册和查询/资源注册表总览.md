## 在 multi-agent 体系中的位置

资源注册表属于 **我的 agents 原理层**，不是单个用例。

它和四层结构的关系是：

- `agent`：看控制层、阶段层、gate 层已经注册了哪些运行态 agent。
- `tool`：看有哪些 `tool_*_operator` 可以被 stage / controller 路由调用。
- `skill`：看哪些能力仍然只是 skill，还没有升级成 registered agent。
- `mcp`：看 Codex 可用的底层服务，例如 Node REPL。
- `plugin`：看 Codex 插件能力，例如 Browser。

所以它适合配合这几篇一起看：

- [[本机已启用agent注册表]]
- [[已启用agent怎么用]]
- [[什么时候把skill做成agent]]
- [[本机资源注册表怎么查]]

## 什么时候打开 HTML 页面

适合打开查询页的场景：

- 想查某个能力到底是 `skill` 还是 `agent`。
- 想确认某个 `tool_*_operator` 有没有被 `config.toml` 注册。
- 想按标签筛选，比如 `dbauto`、`Obsidian`、`蓝鲸`、`浏览器`。
- 想查资源对应的本机源文件路径。
- 想知道某个资源是否已经有 Obsidian 独立卡片。

不适合用它替代的东西：

- 不替代 [[所有agent四层结构和统一流程]]。
- 不替代 [[本机已启用agent注册表]]。
- 不替代每个具体 agent 的职责说明。

它更像“资产清单”和“查询台”。
