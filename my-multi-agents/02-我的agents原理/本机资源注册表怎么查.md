# 本机资源注册表怎么查

## 摘要

现在本机的资源清单不再只靠扫描页面临时展示，而是收敛为一个轻量注册表。

真源：

- `tasks/20260630-agent-catalog/resource_registry.json`

查询页：

- `tasks/20260630-agent-catalog/agent_catalog.html`

默认 Obsidian 地址根：

- `my-multi-agents/02-我的agents原理/本机资源怎么注册和查询/`

当前配置：

- `tasks/20260630-agent-catalog/agent_catalog.config.json`
- `vault_note_root = my-multi-agents/02-我的agents原理/本机资源怎么注册和查询`

- `skill`: 82

注意：HTML 页面上的筛选 tab 是使用口径，不是严格 `kind` 口径。比如 `agent` tab 会把 configured tool agents 一起展示，所以 tab 数量会和这里的真实类型数量略有差异。

## 核心内容

### 当前登记范围

- `skill`
- `agent`
- `tool`
- `mcp`
- `plugin`

### 当前字段

每条资源至少记录：

- `name`
- `desc`
- `source`
- `obsidian`
- `tags`
- `scope`
- `extra`
- `status`

### 使用方式

全量同步并重建页面：

```bash
cd /Users/heytea/Documents/HeyTea/codex-workspace/tasks/20260630-agent-catalog
python3 build_agent_catalog.py
```

手工注册单条资源：

```bash
python3 build_agent_catalog.py register \
  --kind skill \
  --name example-skill \
  --desc "示例 skill" \
  --source /Users/heytea/.codex/skills/example-skill/SKILL.md \
  --obsidian "my-multi-agents/02-我的agents原理/本机资源怎么注册和查询/skill/example-skill.md" \
  --tags "codex,skill,example" \
  --scope "个人 codex skill" \
  --extra "owner=Allen"
```

直接打开查询页：

```bash
/Users/heytea/Documents/HeyTea/codex-workspace/tasks/20260630-agent-catalog/open_agent_catalog.sh
```

### 设计取舍

- 页面只做查询，不做重前端编辑器。
- 注册表用 JSON，不上 SQLite，不加服务端。
- 自动扫描负责补齐现有资源。
- 手工注册负责补齐 `obsidian / scope / extra / tags` 这些人工语义。

## 可执行动作

- 以后新增 `skill / agent / tool / mcp / plugin` 时，同步执行一次 `register`。
- 如果要把某个资源真正沉淀成独立 Obsidian 笔记，就沿用 `my-multi-agents/02-我的agents原理/本机资源怎么注册和查询/<kind>/<name>.md`。
