# 本机资源注册表怎么查

## 摘要

先查生成目录，再回到配置真源，最后运行校验。Obsidian 只提供导航，不承担运行时事实。

## 快速查询

```bash
open ~/.codex/agent-catalog-runtime/agent_catalog.html
```

按资源名、标签或类型筛选后，打开其源文件确认职责和限制。

## 命令行核对

```bash
rg '^\[agents\.' ~/.codex/config.toml
rg '^\[mcp_servers\.' ~/.codex/config.toml
~/.codex/agent-catalog-runtime/check_agent_catalog.sh
```

Agent 是否 active 至少要核对注册项、TOML、未禁用状态和最小验证证据。目标名称只出现在文档中时，仍属于未迁移设计。

## 什么时候用哪个入口

| 问题 | 入口 |
| --- | --- |
| 是否已经有类似能力 | HTML 或 JSON 目录 |
| Agent 是否注册 | `config.toml` + TOML |
| Skill 的实际操作规则 | 对应 `SKILL.md` |
| MCP 是否可用 | MCP 配置与实际工具调用 |
| 生成视图是否漂移 | `check_agent_catalog.sh` |
