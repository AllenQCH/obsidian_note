# Obsidian优秀笔记风格参考

## 摘要

如果你现在觉得 Obsidian 越写越乱，通常不是因为你不会记笔记，而是因为你把**不同类型的内容写成了同一种东西**。

真正好用的 Obsidian 风格，核心不是“排版好看”，而是：

1. 一眼知道这篇笔记是干什么的。
2. 以后找得到、接得上、能继续写。
3. 不会把项目推进、知识沉淀、资料摘录、临时想法混在一起。

所以最值得学的，不是某一个神奇流派，而是下面这几种**各司其职的写法**。

## 核心内容

### 1. MOC / Hub 风格：适合做入口页

#### 这是什么
MOC（Map of Content）本质上就是主题导航页。

#### 典型特征
- 不负责讲细节
- 负责告诉你这个主题下面有什么
- 负责告诉你先看哪篇、后看哪篇
- 常常带阅读顺序、分类目录、重点结论

#### 长什么样
适合写成：
- 主题简介
- 推荐阅读顺序
- 关键子主题
- 相关链接

#### 为什么它好
因为它解决的是：**我明明写过，但后来找不到、也不知道从哪里重新进入。**

#### 适合你当前的场景
- AI 学习路线
- 工具索引
- 项目知识地图
- 某个大主题下已经有 3 篇以上笔记

### 2. Evergreen / 单点知识风格：适合长期复用

#### 这是什么
一篇笔记只回答一个明确问题。

#### 典型特征
- 标题具体，不写“杂记”
- 有一句话结论
- 有“我的理解”
- 会做必要对比和例子
- 可以长期复用、反复链接

#### 长什么样
典型结构：
- 它是什么
- 它解决什么问题
- 我的理解
- 例子 / 对比

#### 为什么它好
因为它让知识从“我当时看过”变成“以后拿出来还讲得清楚”。

#### 适合你当前的场景
- AI 概念理解
- 技术判断
- 工具差异比较
- 某个方法论总结

#### 好标题示例
- `RAG为什么需要Rerank`
- `MCP解决的核心问题`
- `Claude Code 与 Codex 的差异`

#### 坏标题示例
- `AI杂记`
- `一些想法`
- `学习记录`

### 3. Project Dashboard 风格：适合推进事情

#### 这是什么
这类笔记不是为了“沉淀知识”，而是为了“让事情继续往前走”。

#### 典型特征
- 必须有目标
- 必须有当前进展
- 必须有卡点
- 必须有下一步

#### 长什么样
典型结构：
- 目标
- 当前进展
- 决策与约束
- 下一步

#### 为什么它好
因为很多人把项目过程也写成知识笔记，最后就会又长又乱，而且没法继续执行。

#### 适合你当前的场景
- AI 转型计划
- 某个 demo 的开发推进
- OneNote 整理项目
- 某个自动化系统的方案推进

### 4. Resource / Raw 风格：适合先收再整理

#### 这是什么
这是“先保留原始输入”的写法，不要求一开始就写得很成熟。

#### 典型特征
- 清楚标记来源
- 明确说明它还是 raw
- 写清后续要怎么处理
- 不伪装成正式结论

#### 为什么它好
因为它允许你先收集，但不会污染正式知识区。

#### 适合你当前的场景
- 网页剪藏
- OneNote 恢复内容
- 临时摘录
- 会议速记

#### 常见错误
把素材页直接写成结论页。这样最容易让库越来越乱。

### 5. SOP / Workflow 风格：适合反复执行的动作

#### 这是什么
把一个反复要做的事情写成步骤化操作文档。

#### 典型特征
- 面向执行，不面向抒情
- 强调步骤、条件、注意事项
- 最好包含“什么时候用 / 不该怎么用”

#### 为什么它好
因为很多重复动作，本质上不是知识页，而是流程页。

#### 适合你当前的场景
- Obsidian 写作规范
- 知识整理流程
- 某个自动化工具使用方法
- 周报 / 调研 / 会议纪要整理流程

### 6. 真正容易乱的根源

你现在觉得乱，通常是因为下面几种情况：

1. **一篇笔记承担了太多角色**
   - 既是学习记录
   - 又是知识总结
   - 又有项目推进
   - 还混了原始摘录

2. **标题太模糊**
   - 比如“AI杂记”“一些想法”“临时记录”

3. **没有主题入口页**
   - 明明写了很多，却没有一页能把它们串起来

4. **raw 和 processed 没分开**
   - 原始材料和成熟知识混在一起

5. **每篇都想写成百科全书**
   - 结果越写越长，越长越不想回看

### 7. 对你最实用的组合，不是单一流派

如果按你现在的情况，我最推荐的是这套组合：

- **大主题**：用 MOC / Hub
- **核心知识**：用 Evergreen
- **推进中的事**：用 Project Dashboard
- **外部资料与恢复内容**：用 Resource / Raw
- **重复动作**：用 SOP / Workflow

也就是说：

**不是选一种风格统一全库，而是按内容类型选择最合适的风格。**

### 8. 一个最简单、最不容易乱的判断法

以后每次新建笔记前，只先问自己一句：

> 这篇笔记现在最主要的职责是什么？

然后四选一：

- 负责导航 → 索引型
- 负责解释一个问题 → 知识型
- 负责推进一件事 → 项目型
- 负责临时保存输入 → 素材型

只要这一步做对，后面大概率就不会乱。

## 可执行动作

- [ ] 以后新建笔记前，先判断它是索引 / 知识 / 项目 / 素材中的哪一类。
- [ ] 遇到已经写乱的笔记，优先判断“它到底主要应该扮演什么角色”。
- [ ] 对已经超过 3 篇笔记的主题，补一页 MOC / 索引入口。
- [ ] 不再新建 `杂记`、`随手记`、`一些想法` 这类模糊标题。
