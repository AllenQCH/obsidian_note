# dict
使用键-值（key-value）存储，具有极快的查找速度。无序的；需要牢记的第一条就是dict的key必须是**不可变对象**
用dict实现，写“名字”-“成绩”的对照表，直接根据名字查找成绩，无论这个表有多大，查找速度都不会变慢。用Python写一个dict如下：
```plain
>>> d = {'Michael': 95, 'Bob': 75, 'Tracy': 85}
>>> d['Michael']
95
```

```plain
>>> d['Jack'] = 90
>>> d['Jack']
90
>>> d['Jack'] = 88
>>> d['Jack']
88
```

# set
set和dict类似，也是一组key的集合，但不存储value。由于key不能重复，所以，在set中，没有重复的key。set是同一个类型的集合。
1、要创建一个set，用`{x,y,z,...}`列出每个元素：
```plain
方法1:
>>> s = {1, 2, 3}
>>> s
{1, 2, 3}

方法2:
>>> s = set([1, 2, 3])
>>> s
{1, 2, 3}
```

5、set可以看成数学意义上的无序和无重复元素的集合，因此，两个set可以做数学意义上的交集、并集等操作：
```plain
>>> s1 = {1, 2, 3}
>>> s2 = {2, 3, 4}
>>> s1 & s2
{2, 3}
>>> s1 | s2
{1, 2, 3, 4}
```

**区别：set和dict的唯一区别仅在于没有存储对应的value，但是，set的原理和dict一样，所以，同样不可以放入可变对象，因为无法判断两个可变对象是否相等，也就无法保证set内部“不会有重复元素”。试试把list放入set，看看是否会报错。**



# 再议不可变对象
1、对于可变对象，比如list，对list进行操作，list内部的内容是会变化的
```plain
>>> a = ['c', 'b', 'a']
>>> a.sort()
>>> a
['a', 'b', 'c']
```
2、而对于不可变对象，比如str，对str进行操作呢：
```plain
>>> a = 'abc'
>>> a.replace('a', 'A')
'Abc'
>>> a
'abc'
```

**对于不变对象来说，调用对象自身的任意方法，也不会改变该对象自身的内容。
相反，这些方法会创建新的对象并返回，
这样，就保证了不可变对象本身永远是不可变的。**
