## 1) 官方示例：先看这个，最适合你现在

### 仓库

- [GitHub - fastapi/fastapi: FastAPI framework, high performance, easy to learn, fast to code, ready fo](https://github.com/fastapi/fastapi)
    

### 重点不是整个仓库，而是这两个目录

- 入门最小例子： [https://github.com/fastapi/fastapi/tree/master/docs_src/first_steps](https://github.com/fastapi/fastapi/tree/master/docs_src/first_steps)
    
- 稍大一点的项目拆分： [https://github.com/fastapi/fastapi/tree/master/docs_src/bigger_applications/app_an_py310](https://github.com/fastapi/fastapi/tree/master/docs_src/bigger_applications/app_an_py310)
    

### 我为什么推荐它

我查了里面的示例代码，官方给的最小例子就是这种：

```
from fastapi import FastAPI

app = FastAPI()

@app.get("/")

async def root():

    return {"message": "Hello World"}
```

而“大一点的应用”示例已经开始教你：
- `include_router`
- `dependencies`
- `routers/items.py`
- `routers/users.py`
- 模块拆分

这对你现在最有价值，因为你已经学完语法，下一步就是： **“一个 FastAPI 项目到底应该怎么拆文件、拆路由、拆依赖”**
## 2) rednafi/fastapi-nano：最适合拿来“模仿着自己写一遍”

- 仓库：[GitHub - rednafi/fastapi-nano: 🐍 Simple FastAPI template that mimics Flask's blueprint directory str](https://github.com/rednafi/fastapi-nano)
    
- Star：约 **1k+**
### 我看下来它的特点
README 里明确写了：
- 这是一个 **simple FastAPI template**
- 目录结构参考 Flask 的 blueprint 思路
- 带：
- FastAPI CLI / Uvicorn
- Caddy 反向代理
- OAuth2 / JWT
- 密码哈希
### 适合你看的点
这个项目特别适合研究：
- `main.py` 之外的模块怎么组织
- auth 怎么接进去
- 一个“小而完整”的 API 服务长什么样 
### 适合程度
如果你不想一上来被 Docker / 前端 / CI 吓到，**这个非常合适**。

## 3) mahenzon/FastAPI-base-app：轻量版工程骨架

- 仓库：[GitHub - mahenzon/FastAPI-base-app: FastAPI example base application](https://github.com/mahenzon/FastAPI-base-app)
    
- Star：约 **165**
    

### 为什么也值得看

它不是超热门项目，但好处是：

- 比较轻
    
- 是“FastAPI example base application”
    
- 没那么多花活
    

### 适合你研究什么

你可以重点看：

- 项目入口怎么写
    
- 配置怎么管理
    
- 基础目录怎么分
    
- 一个“能跑起来”的后端壳子长什么样
    

如果你觉得大项目看起来头疼，**先拿这个热身**很舒服。

## 首推

### 1) astral-sh/uv-fastapi-example

- 链接：[GitHub - astral-sh/uv-fastapi-example](https://github.com/astral-sh/uv-fastapi-example)
    
- 特点：
    

- **官方 uv 团队（Astral）出的示例**
    
- 用了 **pyproject.toml**
    
- 用了 **uv.lock**
    
- 依赖非常少，核心就是 **FastAPI**
    
- 项目结构很干净，适合拿来学/改成自己的 starter
    

我核对到它的核心配置里就是这种风格：

```
[project]

name = "uv-fastapi-example"

requires-python = ">=3.12"

dependencies = [

    "fastapi[standard]>=0.112.1",

]

```

而且入口代码也非常简单，就是一个标准 FastAPI 多文件结构示例，不是那种“过度工程化”的大模板。

## 如果你现在只想要一个“最简单、最值得抄”的

我建议直接选：

> **astral-sh/uv-fastapi-example**

原因很简单：

1. **官方 uv 示例**
    
2. **技术栈纯正**：`pyproject.toml` + `uv.lock` + `FastAPI`
    
3. **目录少、学习成本低**
    
4. 很适合你后续改成自己的：
    

- `app/main.py`
    
- `app/routers/...`
    
- `pyproject.toml`
    
- `uv.lock`
    

---

## 你可以这样快速开始

如果你本机已经有 `uv`：

```
git clone https://github.com/astral-sh/uv-fastapi-example.git

cd uv-fastapi-example

uv sync

uv run fastapi dev app/main.py

```

然后打开：

- `http://127.0.0.1:8000`
    
- `http://127.0.0.1:8000/docs`
