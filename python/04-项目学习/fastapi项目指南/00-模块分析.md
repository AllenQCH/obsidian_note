# 1、启动项目

因为 uvicorn 启动 ASGI 应用时，吃的不是“文件名随便写”，而是一个固定格式：
<Python模块路径>:<对象名>
正确写法是：uv run uvicorn svc.main:app --reload
```toml
启动流程：进入到项目中并启动
cd /Users/heytea/Documents/New\ project/fastapi-nano
uv run uvicorn svc.main:app --reload --host 127.0.0.1 --port 5002


如果被占用了，杀端口方案；
lsof -i :5002
kill -9 *****
```

# 2、pyproject.toml
这里决定：
它负责：
- 项目元信息
- Python版本
- 依赖管理
- 测试配置
- lint配置
- 类型检查
- 打包配置
重点看：
```toml
dependencies
scripts
tool.uv
```
然后：
```bash
uv sync
uv run uvicorn xxx:app --reload
```
或者：
```bash
pip install -r requirements.txt
```

这里有个截取了pyproject.toml的内容
```toml
dependencies = [ "fastapi[standard]>=0.136.1", "pydantic-settings>=2.14.1", "pwdlib[argon2]>=0.3.0", "pyjwt>=2.12.1", ] 
[dependency-groups] dev = [ "mypy>=2.1.0", "pytest>=9.0.3", "pytest-cov>=7.1.0", "ruff>=0.15.13", ]
```
这些说明了，这些是项目运行必须安装的依赖；类似于maven dependencies
[dependency-groups]   dev = [表示：开发环境依赖
以前全是手写的；现在只需要引入即可；
fastapi[standard]这是FastAPI Web框架；[standard] 指的是安装 FastAPI 推荐全家桶；类似：spring-boot-starter-web
pydantic-settings是配置管理库，管理三方库的环境变量
pwdlib[argon2]是密码加密库，实际上是密码哈希算法
pyjwt是JWT Token库
mypy是Python 类型检查器。类似Java 编译器类型检查
pytest是Python 最主流测试框架。类似JUnit
pytest-cov是测试覆盖率工具。



# 3、项目入口
FastAPI 最核心：
```text
app = FastAPI()
```
所以你先全局搜：
```python
FastAPI(
```
或者：
```python
app = FastAPI()
```
通常会找到：
```text
main.py
app.py
server.py
```
# 4、看路由是怎么挂载的
FastAPI 本质：
```text
路由 -> 函数
```
所以接下来找：
```python
include_router
```
例如：
```python
app.include_router(user_router)
```
这一步你会知道：
- 有哪些模块
- API 怎么拆
- URL结构
# 5、看目录分层
这是最关键的。
你拿到项目后：
先判断它属于哪种架构。
## 小项目
通常：
```text
main.py
routers/
models/
```
## 正规项目（你截图这种）
一般：
```text
apis/      -> controller
core/      -> 基础设施
service/   -> 业务
routes/    -> 路由
tests/     -> 测试
```
## 企业级项目
会继续拆：
```text
domain/
repository/
infra/
application/
```
这时候已经开始：
```text
DDD
```
# 6、看配置系统
Python 项目：
配置非常重要。
重点看：
```text
.env
config.py
settings.py
```
以及：
```python
BaseSettings
```
或者：
```python
os.getenv()
```
你要知道：
- 数据库地址
- Redis
- MQ
- OpenAI key
- 环境变量
# 7、看依赖注入
FastAPI 一个核心：
```text
Depends
```
搜：
```python
Depends(
```
这是：
- 鉴权
- DB Session
- Redis
- 用户信息
注入入口。
类似 Java：
```text
@Autowired
```
# 8、看数据库
搜：
```python
create_engine
AsyncSession
SQLAlchemy
```
或者：
```text
sqlmodel
tortoise
peewee
```
你要知道：
- ORM是什么
- 是否异步
- session怎么管理
- 是否事务
# 9、看核心业务流
这个最重要。
从一个 API 开始。
例如：
```python
@router.post("/order")
```
然后一路跟：
```text
router
 ↓
service
 ↓
repository
 ↓
db
```
你会发现：
FastAPI 本质和 Java Spring：
几乎一样。
# 10、看中间件
搜：
```python
add_middleware
@app.middleware
```
这里决定：
- 日志
- trace
- cors
- jwt
- request_id
# 11、看异步
FastAPI 最大特点：
```text
async
```
所以你要看：
```python
async def
await
```
如果大量 async：
说明：
```text
真正异步项目
```
否则：
可能只是：
```text
伪异步
```
# 12、看启动方式
重点：
```text
docker-compose.yml
Makefile
```
这里通常能看到：
- Redis
- MySQL
- Kafka
- Celery
- Nginx
整个依赖拓扑。
# 13、看测试
搜：
```text
pytest
```
看：
```text
tests/
```
你会知道：
- 是否规范
- 是否企业级
- 是否可维护
# 14、看 AI/RAG/Agent（如果有）
现在很多 FastAPI：
已经不是普通 Web。
搜：
```text
langchain
openai
agent
rag
embedding
vector
chromadb
qdrant
```
这一步能快速判断：
项目是不是 AI 系统。
# 15、最后看整体架构
你最终要总结：
## 这是个什么项目
例如：
```text
FastAPI + SQLAlchemy + Redis
```
## 是同步还是异步
```text
sync / async
```
## 怎么分层
```text
router/service/repository
```
## 怎么部署
```text
docker / k8s
```
## 是否企业级
看：
- 配置
- 日志
- 测试
- middleware
- tracing
- CI/CD

# 一个非常推荐的真实分析顺序
以后你拿到项目：
按这个顺序。
## 第一步
```text
pyproject.toml
README
docker-compose
```
先知道：
```text
怎么跑
```
## 第二步
```text
main.py
FastAPI()
include_router()
```
先知道：
```text
入口
```
## 第三步
随便挑一个 API：
```text
/login
/order
/chat
```
一路 debug。
## 第四步
再看：
```text
config
middleware
db
```
## 第五步
最后才看：
```text
业务代码
```
很多人一上来：
```text
疯狂读代码
```
这是错的。
你应该：
```text
先理解骨架
再理解血肉
```
这和你分析 Java 微服务是一样的。
